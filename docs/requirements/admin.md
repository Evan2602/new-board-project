
## 1. 관리자 활동 로그

### 1.1 관리자 액션 로그
- 관리자의 주요 행위는 모두 로그로 남긴다.
    - 회원 정지 / 해제
    - 비밀번호 초기화
    - 관리자 메모 작성
- 저장 항목:
    - 관리자 ID
    - 액션 타입
    - 대상 사용자 ID
    - 상세 내용
    - 처리 시각

---

## 6. 보안 요구사항

- 관리자 API는 JWT 인증 필수
- 관리자 전용 URL은 별도 Prefix 사용 (/admin/**)
- 관리자 API 접근 시도는 모두 로그로 기록
- 관리자 페이지는 HTTPS 환경에서만 접근 가능 (운영 환경 기준)

---

## 7. 비기능 요구사항

- API 요청 로그 조회 API는 페이징 처리 필수
- 관리자 페이지 주요 화면은 2초 이내 응답 목표
- 로그 테이블 인덱스 설계:
    - requestAt
    - statusCode
    - endpoint(URL)
    - userId
- 대량 로그 조회 시 성능 저하 방지를 위한 인덱스 및 조회 조건 제한

---

## 8. MVP 범위

### 8.1 1차 MVP
- 관리자 로그인
- 회원 목록 / 상세 조회
- 회원 정지 / 해제
- API 요청 로그 조회 (기간 필터 + 상태코드 필터)

### 8.2 2차 확장
- 에러 로그 상세
- Top API 통계
- 관리자 메모 기능
- 관리자 활동 로그

### 8.3 3차 고도화 (선택)
- 이상 트래픽 감지 (5xx 다량 발생 시 알림)
- 로그 CSV 다운로드
- 대시보드 통계 시각화
